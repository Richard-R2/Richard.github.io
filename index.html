GitHub教程 - 分支工作流程

发表于 2018-12-18 | 分类于 GitHub教程 | 评论数： | 阅读次数：
本文字数： 1.2k | 阅读时长 ≈ 1 分钟

1.更新代码

进入代码目录，更新代码

*查看所有远端库情况

*查看所有分支情况，可看到当前的分支是origin/master

2.新建分支

本地新建分支，并切换到新建的分支

3.推送分支

推送新分支到远端库和上游库，冒号前的是本地分支名，冒号后的是远程分支名

此时再敲git branch -a 就可以看到已经在远端库和上游库都建好了分支

4.提交修改

在本地的dev分支修改文件并提交推送到远端库

5.提交PR

在网页通过PR的方式将dev代码推送到上游库的dev分支

6.Fork仓库

等dev调试稳定后再通过PR的方式将上游库的dev分支merge到其master分支，即可上线

  • PR操作可以参考Pull Request 工作流程

GitHub教程 - Pull request 工作流程

发表于 2018-12-18 | 分类于 GitHub教程 | 评论数： | 阅读次数：
本文字数： 595 | 阅读时长 ≈ 1 分钟

1.Fork仓库

先在github上将源仓库fork到自己的仓库

2.检出代码

在本地检出fork后的仓库代码

3.关联上游源仓库

进入代码目录，加入上游源仓库关联

4.更新代码

更新源仓库代码到本地仓库

5.提交代码

在本地仓库中，修改代码后提交到自己的远端仓库

6.提交PR

在github网页上提交PR
在自己的仓库界面点击”New Pull Request”

7.PR合入

然后待源仓库的作者将你的PR合入即可

  • 带分支的工作方式可以参考分支工作流程
